# Session 2 — Time Guide
# The Product-Minded Designer | 2 hours | 6 people

---

## Full timing (120 min)

| Block | Time | What happens |
|-------|------|-------------|
| Opening story (3 slides) | 0:00 – 0:08 | Duolingo story → connect to designers + PMs |
| The chain + the gap | 0:08 – 0:15 | Design decision → consequence chain. The prompt gap. |
| The 5 questions | 0:15 – 0:30 | One slide each. Fast. 2–3 min per question. |
| Case studies | 0:30 – 0:45 | Spotify, Linear, Zomato. 5 min each. |
| **PROBLEM STATEMENT BLOCK** | **0:45 – 1:20** | **This is the core. 35 minutes.** |
| Agent setup + live demo | 1:20 – 1:45 | Set up the system, test on one real brief |
| Resources + close | 1:45 – 2:00 | Share folder, close strong |

---

## Problem Statement Block — How to run it (35 min)

Everyone was asked to come with a problem statement.
This is where you use it.

### Part 1: Share (12 min)
Go around the room. Each person states their problem in 2 sentences.
You are NOT solving it yet. Just listening.
While they talk, mentally note: which one has the most interesting assumptions baked in?

### Part 2: Pick one (2 min)
Pick the one with the most interesting hidden assumption.
Say: "Can I use yours for the live demo? I want to run it through the system with everyone watching."

### Part 3: Live /problem-framer (10 min)
Open Claude Code (or Claude chat if someone doesn't have it).
Run /problem-framer on the chosen problem statement.
Let Claude ask the 5 questions. The owner of the problem answers.
Everyone watches.

The moment Claude flags something the person hadn't thought of — that's the session's best moment.
Don't rush past it. Ask the room: "Did anyone else have this assumption in their own problem?"

### Part 4: Group run (10 min)
Everyone opens Claude on their own laptop.
They run /problem-framer on their own problem statement simultaneously.
Give them 8 minutes to go through it.
Last 2 minutes: "Who got a question from Claude they hadn't thought of? Share it."

### Part 5: Debrief (1 min)
"This is what product thinking looks like in practice.
You didn't change your idea. You changed how you're framing it.
That's the shift."

## For people without Claude Code

They can still do everything:
1. Open claude.ai in the browser
2. Open problem-framer.md from the shared folder
3. Copy the full text, paste it in Claude chat
4. Then paste their problem statement underneath

Same output. Slight more setup. Walk them through it live if needed.

---

## If time runs short

Cut in this order:
1. Trim case studies to 2 (keep Spotify + Zomato, cut Linear)
2. Shorten the 5 questions to 3 (Adjacent Users, North Star, Activation Threshold)
3. Keep the problem statement block — never cut this
4. Skip agent setup; just share the folder and let them explore after

---

## The close (last 2 minutes)

Share the Google Drive folder link in chat.
Say:
"Everything I just showed you is in this folder. The agents are ready to use.
The README tells you exactly what to do.

The only thing I'm asking: Monday morning, before you open Figma,
run /problem-framer on whatever you're working on.
One time. See what happens.

If nothing useful comes out — ignore everything I said today.
If it surfaces one question you hadn't thought of — you'll run it every time."

Then: soft mention of Osiflow.
"I'm building Osiflow HMI — a design platform for teams building interfaces for robots.
If you're curious, the link is in the folder. That's it."
