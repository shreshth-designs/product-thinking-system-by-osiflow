**What:** A Claude-powered product thinking toolkit for designers.
Built by Shreshth Kapoor for the Open Orbit workshop series (Session 2: The Product-Minded Designer).

**Why:** Designers make decisions that cascade into product outcomes, but most designers aren't trained to think about outcomes. This system acts as a PM partner inside Claude.

**Components:**
- 5 slash commands in `.claude/commands/` — run before/during any design work
- 4 agents in `/agents/` — deeper roleplay-based analysis
- `CLAUDE.md` — turns Claude into a product thinking partner by default
- `memory/` — persistent memory across conversations
- `sessions/` — saved outputs from past runs

**Audience:** Designers (individual use + workshop attendees).
**Format:** Shared as a folder; works in Claude Code or Claude.ai browser (copy-paste).

**How to apply:** Treat every question as a product thinking opportunity. Challenge briefs. Ask the 5 questions. Don't validate — interrogate.
