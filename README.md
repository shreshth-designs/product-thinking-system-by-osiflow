## What this is

A product thinking system you keep forever.
5 slash commands. 4 agents. One CLAUDE.md that turns Claude into your PM partner.

Built during a live session. Meant to be used every week.

---

## How to use it

### If you have Claude Code:
1. Open this folder as your project in Claude Code
2. Type any /command below

### If you don't have Claude Code:
1. Open Claude.ai in your browser
2. Open any file from .claude/commands/ or agents/
3. Copy the full contents
4. Paste into Claude chat, then paste your brief underneath

---

## The 5 commands (run in this order)

| Command | When to use |
|---------|-------------|
| /problem-framer | Before designing anything |
| /brief-interrogator | When you have a brief to check |
| /adjacent-user | When you're stuck on who to design for |
| /tradeoff | When deciding between two directions |
| /adversarial-review | Before handing anything off |

## The 4 agents (open from /agents folder)

| Agent | What it does |
|-------|-------------|
| pm-reviewer.md | A senior PM tears apart your design decision |
| user-researcher.md | Simulates a real user interview |
| devils-advocate.md | Finds everything that could go wrong |
| product-strategist.md | Big picture — is this a feature or a moat? |

---

## The right order for any design problem

1. /problem-framer — what are we actually solving?
2. /brief-interrogator — is the brief PM-quality?
3. /adjacent-user — who are we forgetting?
4. Run pm-reviewer agent — how does a PM see this?
5. /tradeoff — what are we trading off?
6. /adversarial-review — what breaks before it ships?
